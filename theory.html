Теориялық бөлім

Бұл бөлімде Web-жобалаудың 4 негізгі сабағы берілген. Әр сабақта түсіндірме, мысал және қысқа тапсырмалар арқылы білімді бекітесіз.

Ереже
HTML

Тегтерді дұрыс жабу және мағыналы (семантикалық) тегтерді қолдану бетті түсінікті етеді.

Кеңес
CSS

JS

Интерактивті шағын қадаммен бастаңыз: батырмаға click оқиғасын қосып, мәтінді өзгертіңіз.

Веб қалай жұмыс істейді?

Веб-бетті ашқанда браузер серверге сұраныс жібереді. Сервер жауап ретінде HTML, CSS, JavaScript және медиа файлдарды береді. Браузер оларды біріктіріп, экранға шығарады. Бұл процесс клиент–сервер байланысына негізделген.

Браузер алдымен домен атын сервердің IP-мекенжайына сәйкестендіру үшін DNS жүйесін қолданады. Содан кейін HTTPS болса, қауіпсіз байланыс орнату үшін шифрлау келісімі (TLS) орындалады. Осыдан кейін ғана HTTP сұранысы жіберіліп, серверден жауап келеді. Жауап құрамында тек бет мәтіні емес, қосымша файлдарға сілтемелер де болады.

HTML — бет құрылымының негізі болғандықтан, браузер оны оқи салысымен DOM ағашын құрастырады. CSS файлдары келгенде браузер стиль ережелерін есептеп, элементтердің нақты өлшемі мен орнын анықтайды. JavaScript орындалғанда беттегі кейбір элементтер өзгеруі, жаңа мәлімет жүктелуі немесе пайдаланушыға хабарлама шығуы мүмкін. Сондықтан бір сайттың жұмысы — бірнеше файлдың бірлескен нәтижесі.

Кейде бет тез ашылады, себебі браузер бұрын жүктелген файлдарды кэште сақтайды. Бірақ кэш тым ескі болса, жаңа өзгерістер көрінбеуі мүмкін — онда парақты жаңарту қажет. Веб-жобалауда файл атауын түсінікті қою, құрылымды реттеу және қажетсіз үлкен суреттерді қолданбау маңызды. Бұл пайдаланушыға ыңғайлы әрі жылдам сайт жасауға көмектеседі.

Front-end және Back-end (қысқаша)

Веб-жобада екі негізгі бағыт бар: front-end және back-end. Front-end — пайдаланушы көретін бөлік: интерфейс, батырмалар, мәтін, дизайн және интерактив (HTML/CSS/JS). Back-end — сервер жағындағы логика: дерекқор, тіркелу, есептеу, файл сақтау сияқты қызметтер. Көп сайтта бұл екі бөлік бірге жұмыс істейді: front-end сұрайды, back-end жауап береді.

Мектеп деңгейінде сіз ең алдымен front-end негізін үйренесіз, себебі оны браузерде бірден көріп тексеруге болады. Дегенмен қауіпсіздік пен этика қағидаларын да есте ұстау керек: жеке мәліметті жарияламау, күмәнді сілтемелерді ашпау және авторлық құқықты сақтау. Мысалы, сурет қолдансаңыз, оның ашық лицензиясы бар-жоғын тексерген дұрыс. Бұл да — жауапты веб-жобалаудың бір бөлігі.

Есте сақта: Семантика не үшін маңызды?
Семантикалық HTML (мысалы: <header>, <main>, <article>, <footer>) мазмұнның мағынасын көрсетеді. Бұл қолжетімділікке, іздеу жүйесіне және браузердің дұрыс жұмыс істеуіне көмектеседі.
Цифрлық қауіпсіздік: жеке дерек енгізетін сайттарда HTTPS және сенімді доменге назар аударыңыз. Бейтаныс файлдарды жүктемес бұрын мұғалімнен/ата-анадан сұраған дұрыс.

HTML құжатының “қаңқасы”

HTML құжат <!doctype html> жолынан басталады — бұл браузерге құжатты стандарт режимде көрсетуге көмектеседі. Одан кейін <html>, <head>, <body> бөлімдері болады. <head> — қызметтік ақпарат (атау, кодтау, стиль/скрипт), <body> — экрандағы мазмұн.

HTML-де әр элемент ашылу және жабылу тегтерімен жазылады, бірақ кейбірі өздігінен жабылады (мысалы: <img>, <br>). Тегтердің ішіне салыну реті маңызды: бір элемент екіншісінің ішінде дұрыс жабылуы тиіс. Қателессеңіз, бетте күтпеген орналасу немесе стиль мәселесі пайда болуы мүмкін. Сондықтан кодты ұқыпты жазу — жақсы әдет.

Семантикалық тегтерді қолданғанда мәтіннің мағынасы анық көрінеді: <nav> — навигация, <main> — негізгі мазмұн, <section> — бөлім, <footer> — төменгі ақпарат. Бұл тек әдемілік үшін емес: экран оқитын бағдарламаларға (қолжетімділік) және іздеу жүйесіне пайдалы. Яғни сайттың сапасы артады.

Файлдарды реттеу де маңызды: мысалы, суреттерді assets қалтасына, стильді styles.css файлына, ал логиканы app.js файлына бөліп сақтаған дұрыс. Осылайша жобаны түсіну жеңіл болады, әрі болашақта өзгерту тезірек орындалады. Мектеп жобасында да “тәртіп” — нәтиженің кепілі.

Сурет пен сілтемені дұрыс қолдану

Сурет қойған кезде оның өлшемін алдын ала ойлаңыз: өте үлкен файл сайтты баяулатады. Әр суретке alt жазу керек, себебі сурет ашылмаса немесе экран оқитын құрал қолданылса, сол мәтін оқылады. Сілтеме (<a>) мәтіні қысқа әрі нақты болсын: мысалы, “Тесттер бөлімі” немесе “Кері байланыс” сияқты.

Егер сыртқы сайтқа өтсеңіз, сенімді ресурстарды таңдаңыз және күмәнді жарнамалық сілтемелерден сақ болыңыз. Мектеп тапсырмаларында авторлық құқықты сақтаңыз: интернеттен алынған суретке дереккөз көрсету жақсы мәдениет. Осының бәрі — жауапты цифрлық азамат болудың белгісі.

Қолжетімділік (Accessibility) бойынша шағын ереже
  • Тақырыптарды ретімен қолданыңыз: h1 → h2 → h3…
  • Суретке alt жазыңыз: мағынасы бар суретте alt міндетті.
  • Сілтемені түсінікті атаңыз: “Мына жерді бас” емес, нақты атау.

Практикаға жақын тапсырма

1) Өз бетіңіз туралы шағын мәтін жазыңыз (h1, p, ul). 2) Бір сурет қосыңыз және alt толтырыңыз. 3) Екі ішкі сілтеме жасаңыз: “Глоссарий” және “Тесттер” бетіне. 4) Кодты сақтап, браузерде ашып тексеріңіз. Осы әрекеттер HTML құрылымын жақсы бекітеді.

CSS ережесінің құрылымы

CSS ережесі селектор + қасиет: мән жұбынан тұрады: p { color: blue; } — барлық абзацтың түсін өзгертеді. Селектор ретінде элемент атауы, .class, #id жиі қолданылады.

CSS-ті үш түрлі жолмен қосуға болады: 1) сыртқы файл арқылы (link), 2) <style> ішінде, 3) элементтің өзіне тікелей жазу (inline style). Мектеп жобаларында ең дұрыс тәсіл — сыртқы styles.css файлын пайдалану. Себебі барлық стиль бір жерде тұрады, қате табу және өзгерту жеңіл болады.

Каскадтың мәнін түсіну үшін мысал: бір элементке p {color: blue;} және .warning {color: red;} қатар жазылса, класс нақтылығы жоғары болғандықтан мәтін қызыл болады. Ал егер !important қолдансаңыз, ол көбіне басқа ережелерден басым түседі, бірақ оны жиі қолдануға болмайды. Өйткені кодты басқару қиындайды және қателік көбейеді.

Орналасу жасағанда Flexbox көбіне “бір қатар” немесе “бір баған” логикасына ыңғайлы, ал Grid — кесте сияқты тор жүйесіне жақсы. Мысалы, мәзірді Flexbox-пен, ал карточкалардан тұратын бөлімді Grid-пен жасау тиімді. Осы екі құралды дұрыс таңдау — кәсіби веб-жобалаудың негізі.

Оқылымдылық және дизайн мәдениеті

Мектеп оқулығы стилінде дизайн ең алдымен оқуға ыңғайлы болуы тиіс: мәтін анық, аралығы жеткілікті, фон көзге жайлы болуы керек. Қаріптің тым ұсақ болмауын және мәтін мен фонның контрастын сақтаған дұрыс. Түстерді көп қолданбай, 2–3 негізгі түспен шектелсеңіз, бет жинақы көрінеді. Сондай-ақ ұзын мәтінді абзацтарға бөліп, тақырыптарды айқын етіп қойыңыз.

Анимацияны да орнымен қолдану керек: өте жылдам немесе көп қозғалыс оқушының назарын бөледі. Сол себепті ұсақ ауысу эффектілері (fade, hover) жеткілікті. Және prefers-reduced-motion параметрін ескерсеңіз, қозғалысқа сезімтал пайдаланушыларға да ыңғайлы болады.

Алгоритм ойлау: шағын мысал

Мысалы, “батырма басылса — ұпай 1-ге артсын” деген ереже құрайық. Бұл жерде 1) бастапқы ұпайды let score = 0; деп қоямыз, 2) батырманы табамыз, 3) click оқиғасын тыңдаймыз, 4) оқиға болғанда ұпайды арттырып, экрандағы мәтінді жаңартамыз. Осындай қадамдар — алгоритмдік ойлаудың қарапайым үлгісі.

Қорытынды тапсырма

Өзіңіздің шағын “Калькулятор” немесе “Санағыш” бетіңізді құрастырыңыз: екі батырма (арттыру/азайту), бір мәтін, және нәтижені шығаратын блок болсын. Қате шықса, Console арқылы тексеріңіз. Дайын болған соң, беттер арасындағы ауысу анимациясының қалай жұмыс істейтінін байқап көріңіз.

Қорытынды: JavaScript — веб-бетті “тірілтетін” құрал. Айнымалы, функция және оқиға арқылы интерактив құра аласыз.

JavaScript-те дерек түрлері бар: сан (number), мәтін (string), логикалық мән (boolean), массив (array) және объект (object). Мысалы, оқушының ұпайын сан ретінде, ал аты-жөнін мәтін ретінде сақтауға болады. Осы деректерді салыстыру үшін шарт операторлары қолданылады: if, else. Қайталау керек болса, циклдер көмектеседі: for немесе while.

Оқиға өңдеуші (event listener) — “құлақ түріп тұратын” код бөлігі. Мысалы, оқушы мәтін тергенде input оқиғасы, ал батырма басқанда click оқиғасы іске қосылады. Осы кезде сіз DOM-дағы элементтің мәтінін, түсін, көрінуін өзгерте аласыз немесе есептеу жүргізесіз. Сондықтан JS — вебті интерактив ететін негізгі құрал.

Қауіпсіздік жағынан маңызды ереже бар: пайдаланушы енгізген мәтінді “сол күйі” HTML ретінде қоспау керек. Сол үшін көбіне textContent қолдану қауіпсіз, ал innerHTML абайлап қолданылуы тиіс. Мектеп жобаларында да қауіпсіздік дағдысын үйрену пайдалы. Бұл — болашақта үлкен жобалар жасағанда көмектесетін мәдениет.
